Boas Práticas de Código em Python

Por que seguir Boas Práticas?

Seguir boas práticas de código torna o desenvolvimento mais eficiente e o código mais legível e fácil de manter.

Em Python, o guia de estilo mais adotado é o PEP 8, que define convenções para o código Python.

Nomenclatura de Variáveis e Funções

Use nomes descritivos para variáveis e funções, para que o código seja claro e fácil de entender.

  • Para variáveis e funções, use letras minúsculas e underscores para separar palavras (minha_variavel, calcular_media).
  • Para constantes, use letras maiúsculas (TAMANHO_MAXIMO).

Identação

Em Python, a identação é obrigatória e ajuda a definir blocos de código. Use 4 espaços para cada nível de identação.

def minha_funcao():
    if True:
        print("Exemplo de identação")

Linhas em Branco

Use linhas em branco para separar blocos de código e tornar o código mais organizado.

  • Deixe duas linhas em branco entre definições de funções e classes.
  • Dentro de funções, use uma linha em branco para separar diferentes partes da lógica.

Comentários

Comente seu código para explicar partes complexas e documentar o propósito de funções e variáveis.

  • Use comentários de uma linha com # para explicar o código.
  • Para docstrings, use aspas triplas (""") no início de funções, métodos e classes para descrever seu propósito.
def calcular_area(raio):
    """Calcula a área de um círculo dado o raio."""
    return 3.14 * raio ** 2

Evite Código Repetido

Se encontrar código repetido, considere transformá-lo em uma função. Isso ajuda a evitar redundância e torna o código mais modular.

def calcular_area(raio):
    return 3.14 * raio ** 2

def calcular_volume(raio, altura):
    return calcular_area(raio) * altura

Neste exemplo, a função calcular_area é reutilizada dentro de calcular_volume.

Documentação

Mantenha a documentação do seu código atualizada, especialmente se outras pessoas irão trabalhar com ele.

Docstrings bem escritas em classes e funções ajudam a entender o funcionamento do código.

Tratamento de Erros

Use blocos try e except para capturar erros e evitar que o programa falhe inesperadamente.

def dividir(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        print("Erro: divisão por zero não é permitida.")
        return None

Neste exemplo, o erro de divisão por zero é capturado e tratado de forma apropriada.

Exemplo Completo

Veja um exemplo de código que segue boas práticas:

def calcular_media(notas):
    """Calcula a média de uma lista de notas."""
    if not notas:
        return 0
    soma = sum(notas)
    return soma / len(notas)

# Lista de notas de um aluno
notas_aluno = [7.5, 8.0, 6.5, 9.0]

# Calcula a média e imprime o resultado
media = calcular_media(notas_aluno)
print(f"Média do aluno: {media:.2f}")

Este exemplo segue as boas práticas: usa nomes descritivos, inclui uma docstring, evita redundância e formata o resultado corretamente.
