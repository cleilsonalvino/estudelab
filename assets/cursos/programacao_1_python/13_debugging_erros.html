Debugging e Tratamento de Erros

O que é Debugging?

Debugging é o processo de identificar e corrigir erros em um programa. Essa prática é fundamental para garantir que o código funcione corretamente.

No Python, podemos usar várias ferramentas e técnicas para facilitar o debugging.

Tipos Comuns de Erros

  • Erros de Sintaxe: Ocorrem quando o código está malformado ou faltam elementos, como parênteses ou pontos e vírgulas.
  • Erros de Execução: Erros que ocorrem enquanto o programa é executado, como divisão por zero ou acesso a uma variável inexistente.
  • Erros Lógicos: Ocorrem quando o código roda sem erros, mas o resultado não é o esperado devido a uma lógica incorreta.

Tratamento de Erros com Try e Except

Usar blocos try e except permite capturar e tratar erros sem que o programa pare de funcionar.

try:
    numero = int(input("Digite um número: "))
    resultado = 10 / numero
    print("Resultado:", resultado)
except ZeroDivisionError:
    print("Erro: Não é possível dividir por zero.")
except ValueError:
    print("Erro: Entrada inválida, digite um número.")

Neste exemplo, são tratados erros de divisão por zero e erros de entrada inválida.

Usando o Finally

A cláusula finally é executada independentemente de ocorrer um erro ou não, sendo útil para garantir que determinadas ações sempre aconteçam.

try:
    arquivo = open("dados.txt", "r")
    dados = arquivo.read()
except FileNotFoundError:
    print("Erro: Arquivo não encontrado.")
finally:
    arquivo.close()
    print("Arquivo fechado.")

O bloco finally garante que o arquivo seja fechado, independentemente de haver um erro.

Ferramentas para Debugging

Python possui algumas ferramentas e funções integradas que ajudam a inspecionar o código durante o debugging:

  • print(): A função print() é útil para exibir valores de variáveis em diferentes pontos do código, ajudando a identificar onde está o erro.
  • pdb: O módulo pdb é o depurador padrão do Python. Ele permite executar o código passo a passo e inspecionar variáveis.

Exemplo com pdb

Para usar o pdb, basta importar o módulo e chamar pdb.set_trace() onde deseja iniciar a depuração.

import pdb

def soma(a, b):
    pdb.set_trace()  # Inicia o modo de depuração aqui
    return a + b

resultado = soma(5, "10")  # Provoca um erro de tipo

Com pdb, você pode inspecionar variáveis e avançar no código linha por linha, ajudando a identificar o erro.

Boas Práticas para Debugging

  • Use comentários: Adicione comentários para entender melhor o fluxo do código.
  • Teste partes do código separadamente: Teste funções e blocos individualmente para identificar a fonte do erro.
  • Revise a lógica: Verifique se a lógica implementada está de acordo com o esperado.
  • Exiba valores com print(): Use print() para mostrar o valor de variáveis durante a execução e verificar se estão corretos.

Exemplo Completo

Veja um exemplo completo que inclui tratamento de erros e debugging com print():

def dividir(a, b):
    try:
        print("Dividendo:", a)  # Exibe o valor do dividendo
        print("Divisor:", b)    # Exibe o valor do divisor
        resultado = a / b
        return resultado
    except ZeroDivisionError:
        print("Erro: Divisão por zero.")
        return None
    except TypeError:
        print("Erro: Tipo inválido. Certifique-se de passar números.")
        return None

# Teste da função com entrada incorreta
resultado = dividir(10, 0)
print("Resultado:", resultado)

Este exemplo mostra como usar print() para verificar valores e como tratar erros comuns, como divisão por zero e tipo inválido.
